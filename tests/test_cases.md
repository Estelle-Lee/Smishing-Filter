# 테스트 케이스

## 스미싱 예시 (위험)

### 1. 택배 사칭
```
[Web발신]
택배가 도착했습니다.
확인: https://bit.ly/xxxxx
```

### 2. 은행 사칭
```
[KB국민은행]
고객님의 보안카드가 만료되었습니다.
긴급 재등록: http://kb-update.com
```

### 3. 정부기관 사칭
```
[국세청]
미납 세금이 있습니다.
24시간 내 납부하지 않으면 법적 조치
납부하기: https://me2.do/xxxxx
```

### 4. 쇼핑몰 사칭
```
[쿠팡]
주문하신 상품이 반송되었습니다.
재배송 신청: http://coupang-cs.net
```

### 5. 스마트폰 당첨
```
축하합니다!
갤럭시 S24 당첨되셨습니다.
수령: https://tinyurl.com/xxxxx
```

## 정상 문자 예시 (안전)

### 1. 정상 은행 문자
```
[KB국민은행]
05/15 12:34
체크카드 500,000원 승인
잔액 1,234,567원
```

### 2. 정상 택배 문자
```
[CJ대한통운]
송장번호: 123456789012
배송이 시작되었습니다.
조회: https://www.cjlogistics.com
```

### 3. 가족/지인
```
엄마
저녁 먹고 와
국 끓여놨어
```

### 4. 인증번호
```
[네이버]
인증번호: [123456]
타인 노출 금지
```

### 5. 예약 확인
```
[카카오헤어샵]
5월 20일 오후 2시
홍길동 고객님 예약 확인
취소는 1일 전까지 가능
```

## 테스트 방법

1. 위 텍스트를 복사하여 Streamlit 앱의 "텍스트 입력" 탭에 붙여넣기
2. 스미싱 예시는 높은 위험도가 나와야 함
3. 정상 문자는 낮은 위험도가 나와야 함
4. 각 케이스별로 적절한 이유와 가이드가 제공되는지 확인

## 스크린샷 준비 팁

1. 실제 스미싱 문자 받은 게 있으면 스크린샷 찍기
2. 구글 검색: "스미싱 문자 예시"
3. 뉴스 기사에서 스미싱 이미지 캡처
4. 테스트용으로 위 텍스트를 메모장에 입력 후 스크린샷

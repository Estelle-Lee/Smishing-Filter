## 🎤 발표 스크립트 (4분)

### 1분: 문제 제시
> "여러분, 작년 한 해 스미싱 피해액이 500억 원을 넘었습니다. 이 중 60%가 60대 이상 어르신들입니다. '택배가 도착했습니다' 같은 간단한 문자 하나에 평생 모은 돈을 잃는 분들이 계십니다."

### 1분: 솔루션 소개
> "저는 이 문제를 해결하기 위해 '스미싱 지킴이'를 만들었습니다. AI가 의심스러운 문자를 자동으로 분석해서 위험도를 알려주는 서비스입니다."

### 2분: 실시간 데모
1. **스미싱 문자 분석**
   - 화면 공유: 앱 실행
   - 텍스트 입력: "택배 도착 http://bit.ly/xxxxx"
   - 결과: "위험도 95% - 스미싱입니다"

2. **정상 문자 분석**
   - 텍스트 입력: "KB국민은행 잔액 확인"
   - 결과: "위험도 10% - 안전합니다"

3. **스크린샷 분석** (시간 있으면)
   - 실제 스미싱 이미지 업로드
   - 결과 확인

### 30초: 차별화 포인트
> "기존 필터링과의 차이점은 세 가지입니다. 첫째, GPT-4 Vision을 사용해 스크린샷도 분석 가능합니다. 둘째, 한국어 특화 프롬프트로 정확도가 높습니다. 셋째, API로 다른 앱에 쉽게 통합할 수 있습니다."

### 30초: 확장 가능성
> "현재는 데모 앱이지만, API를 제공하면 은행 앱, 통신사 앱에 통합할 수 있습니다. 또한 카카오톡, 텔레그램 같은 메신저 플랫폼에도 적용 가능합니다."

## 💡 예상 질문 & 답변

### Q: "네이버나 카카오도 이미 필터링하는데 차별점이 뭔가요?"
**A**: "대기업 서비스는 자사 플랫폼만 커버합니다. 저희는 API로 제공해서 중소 스타트업, 커뮤니티 사이트도 쉽게 도입할 수 있습니다. 또한 스크린샷 분석 기능으로 사용자가 직접 의심 문자를 체크할 수 있습니다."

### Q: "GPT-4 API 비용이 비싸지 않나요?"
**A**: "텍스트 분석은 건당 약 3원, 이미지 분석도 10-20원 수준입니다. 월 10만 건 분석해도 100-200만 원으로 충분히 수익 모델이 가능합니다."

### Q: "정확도는 얼마나 되나요?"
**A**: "현재 테스트 케이스 기준 약 90%입니다. 실제 서비스에서는 사용자 피드백으로 지속 개선할 계획입니다. 오탐률을 낮추기 위해 '의심' 등급을 세분화했습니다."

### Q: "개인정보 보호는 어떻게 하나요?"
**A**: "분석은 실시간으로 이루어지고 문자 내용은 저장하지 않습니다. OpenAI API도 30일 후 자동 삭제 정책을 따릅니다."

### Q: "수익 모델은?"
**A**: "B2B SaaS 모델입니다. 기업용 API 구독 (월 10만 원~) + 건당 과금. 초기에는 무료 티어로 개발자 유입 후 유료 전환 예정입니다."
